## 一、问题概述

### 问题 #1: 签名预密钥未存储在本地

**原始问题**:
- 签名预密钥 (`signedPrekeys`) 未存储在 localStorage 中
- 只有身份密钥对 (`identityKeyPair`) 存在
- 导致无法进行签名验证测试

**严重程度**: 🟡 中

---

## 二、根本原因分析

### 原因 1: IdentityKeyPair 类型不完整

`IdentityKeyPair` 类型缺少 `signingPrivateKey` 和 `signingPublicKey` 字段，导致无法正确签名预密钥。

```typescript
// 修复前
export interface IdentityKeyPair {
  privateKey: CryptoKey;
  publicKey: CryptoKey;
  publicKeyBytes: Uint8Array;
}

// 修复后
export interface IdentityKeyPair {
  privateKey: CryptoKey;  // ECDH 私钥，用于密钥交换
  publicKey: CryptoKey;   // ECDH 公钥
  publicKeyBytes: Uint8Array;
  signingPrivateKey: CryptoKey;  // ECDSA 私钥，用于签名
  signingPublicKey: CryptoKey;   // ECDSA 公钥，用于验证签名
}
```

### 原因 2: Web Crypto API 限制

Web Crypto API 不允许 ECDH 密钥同时用于密钥交换和 ECDSA 签名：
- ECDH 密钥只能用于 `['deriveBits', 'deriveKey']`
- ECDSA 密钥只能用于 `['sign', 'verify']`

尝试同时使用会导致错误：
```
InvalidAccessError: key.algorithm does not match that of operation
```

### 原因 3: KeyManager 实例不一致

`use-signal.ts` 和 `messageEncryptionService` 使用了不同的 `KeyManager` 实例，导致预密钥保存到一个实例的存储中，但检查时使用的是另一个实例。

### 修复 2: 生成独立的签名密钥对

**文件**: `apps/desktop/src/core/signal/index.ts`

```typescript
static async generateIdentityKeyPair(): Promise<IdentityKeyPair> {
  // ECDH 密钥对用于密钥交换
  const ecdhKeyPair = await crypto.subtle.generateKey(
    { name: 'ECDH', namedCurve: 'P-256' },
    true,
    ['deriveBits', 'deriveKey']
  );

  // ECDSA 密钥对用于签名
  const signingKeyPair = await this.generateSigningKeyPair();

  const publicKeyBytes = await crypto.subtle.exportKey('raw', ecdhKeyPair.publicKey);

  return {
    privateKey: ecdhKeyPair.privateKey,
    publicKey: ecdhKeyPair.publicKey,
    publicKeyBytes: new Uint8Array(publicKeyBytes),
    signingPrivateKey: signingKeyPair.privateKey,
    signingPublicKey: signingKeyPair.publicKey,
  };
}
```

### 修复 3: 使用签名私钥进行签名

**文件**: `apps/desktop/src/core/signal/index.ts`

```typescript
static async generateSignedPrekey(identityKeyPair: IdentityKeyPair, keyId: number): Promise<SignedPrekey> {
  const keyPair = await crypto.subtle.generateKey(
    { name: 'ECDH', namedCurve: 'P-256' },
    true,
    ['deriveBits', 'deriveKey']
  );

  // 使用身份签名私钥对签名预密钥公钥进行签名
  const publicKeyBytes = await crypto.subtle.exportKey('raw', keyPair.publicKey);
  const signature = await crypto.subtle.sign(
    { name: 'ECDSA', hash: 'SHA-256' },
    identityKeyPair.signingPrivateKey,  // 使用 signingPrivateKey 而不是 privateKey
    publicKeyBytes
  );

  return {
    keyId,
    keyPair,
    signature: new Uint8Array(signature),
  };
}
```
