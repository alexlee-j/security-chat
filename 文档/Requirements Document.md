# 全平台通讯APP需求文档

（普通聊天 + 阅后即焚 | RN + Tauri 单工程架构）

---

# 1. 文档概述

## 1.1 文档目的

本文档明确全平台通讯APP（支持 iOS、Android、macOS、Windows）的核心需求、功能细节、跨平台架构规范、安全模型、性能指标及验收标准。

本产品采用：

* React Native（移动端）
* Tauri（桌面端）
* Monorepo 单代码仓库
* NestJS + WebSocket 后端架构

确保：

* 单工程维护
* 跨平台一致性
* 安全通讯能力
* 可扩展架构设计

---

## 1.2 产品定位

一款强调：

* 高隐私保护
* 跨设备同步
* 简洁易用
* 端到端加密

的即时通讯应用。

对标产品能力级别（参考安全模型）：

* Signal（安全模型）
* Telegram（多端同步）
* WhatsApp（大众化体验）

但产品定位为：

> 轻量 + 安全优先 + 全平台统一体验

---

## 1.3 技术架构定位

### 前端

* React Native（iOS / Android）
* Tauri + React（macOS / Windows）
* 共享 UI 组件与业务逻辑

### 后端

* NestJS
* PostgreSQL
* Redis
* WebSocket Gateway
* 对象存储（图片 / 文件）

---

# 2. 核心用户需求

## 2.1 用户角色

目标用户：

* 18-45 岁
* 多设备办公 / 社交用户
* 对隐私保护有明确需求

---

## 2.2 核心需求

1. 支持四平台安装使用
2. 登录同账号可多端同步
3. 普通聊天流畅稳定
4. 阅后即焚严格隐私销毁
5. 操作简单统一

---

# 3. 系统架构设计（技术级需求）

---

## 3.1 Monorepo 结构

```
apps/
  mobile/      (React Native)
  desktop/     (Tauri)
  backend/     (NestJS)

packages/
  ui/
  crypto/
  websocket/
  shared-types/
  state/
```

---

## 3.2 通讯协议

* WebSocket 长连接
* TLS 1.3 加密
* JSON 协议结构

---

## 3.3 加密架构（核心）

采用双层安全模型：

1. 传输层：TLS 1.3
2. 消息层：端到端 AES-256 加密

密钥管理：

* 每个会话独立密钥
* 本地安全模块存储：

  * iOS Keychain
  * Android Keystore
  * macOS Keychain
  * Windows Credential Vault

---

# 4. 详细功能需求

---

# 4.1 用户系统

### 注册

* 手机号 + 验证码
* 邮箱 + 验证码
* 密码强度校验

* 搜索用户

* 备注 / 分组
* 黑名单
* 在线状态

同步延迟 ≤ 3 秒

---

# 4.3 普通聊天功能

---

## 支持消息类型

* 文本（≤5000字）
* 图片（≤9张）
* 文件（≤100MB）
* 语音（≤60秒）
* 表情
* 视频通话 / 语音通话

---

## 消息状态

* sending
* sent
* delivered
* read

---

## 多端同步规则

* 普通聊天参与同步
* 已读状态实时同步
* 离线自动补偿同步

---

# 4.4 阅后即焚功能（核心安全模块）

---

## 4.4.1 模式机制

* 单条消息可设置
* 会话级默认开启
* UI 明显红色标识

---

## 4.4.2 支持类型

* 文本
* 图片
* 语音

不支持：

* 文件
* 视频通话

---

## 4.4.3 销毁规则

| 类型 | 销毁触发      |
| -- | --------- |
| 文本 | 打开聊天框开始计时 |
| 图片 | 打开预览开始计时  |
| 语音 | 播放开始计时    |

支持时长：

* 5秒
* 10秒
* 30秒
* 1分钟
* 5分钟

---

## 4.4.4 安全控制

### 禁止行为

* 转发
* 复制
* 保存
* 多次播放

---

### 截屏限制

移动端：

* Android 使用 secure flag
* iOS 截屏监听

桌面端：

* 截屏行为提示
* 强制水印

---

## 4.4.5 异常规则

* 网络中断暂停计时
* 超过24小时未查看自动销毁
* 所有设备同步删除

---

# 4.5 通知系统

移动端：

* iOS 使用 APNs
* Android 使用 FCM

桌面端：

* 系统通知 API

阅后即焚通知：

> 仅显示“收到一条阅后即焚消息”

不展示内容摘要

# 7. 安全要求（强制）

---

## 数据存储

* 本地数据库加密（SQLCipher）
* 服务器端加密存储

---

## 端到端加密

服务端：

* 不存储明文
* 不持有会话密钥

---

## 防攻击机制

* 防重放攻击
* 防暴力破解
* Token 黑名单机制
* WebSocket 心跳检测

---

# 8. 合规要求

* 遵守各应用商店规范
* 隐私政策明确说明
* 支持数据删除权

---

# 9. 验收标准

---

## 功能验收

* 普通聊天稳定
* 阅后即焚无留存
* 多端同步无错乱

---

## 安全验收

* 抓包无法看到明文
* 本地数据库不可直接读取
* 阅后即焚销毁后不可恢复
