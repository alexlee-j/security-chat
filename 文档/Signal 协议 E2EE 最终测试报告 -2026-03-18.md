## 🔧 修复和优化的功能

### 1. 注册流程生成 Signal 密钥

**修复内容**:
- 注册时自动生成身份密钥对
- 生成签名预密钥（使用身份私钥签名）
- 生成 100 个一次性预密钥
- 自动上传预密钥到服务器

**文件**:
- `apps/desktop/src/core/use-chat-client.ts`

### 2. 预密钥本地存储

**优化内容**:
- 签名预密钥正确存储到 localStorage
- 一次性预密钥正确存储到 localStorage
- 添加预密钥状态检查函数
- 添加预密钥自动补充机制

**文件**:
- `apps/desktop/src/core/use-signal.ts`
- `apps/desktop/src/core/signal/key-management.ts`

### 3. 签名预密钥使用真实签名

**修复内容**:
- 使用身份私钥对签名预密钥公钥进行 ECDSA 签名
- 签名可以被身份公钥验证

**文件**:
- `apps/desktop/src/core/signal/index.ts`

### 4. Double Ratchet 完善

**优化内容**:
- 实现完整的 DH 棘轮
- 实现 HKDF 密钥派生函数
- 支持前向保密和后向保密

**文件**:
- `apps/desktop/src/core/signal/double-ratchet.ts` (新建)

### 5. 预密钥包访问控制

**修复内容**:
- 只有好友或自己可以获取预密钥包
- 防止未授权用户获取密钥材料

## 🐛 发现的问题和解决

### 问题 1: Token 存储方式

**发现**: Token 没有存储在 localStorage 的明文位置，而是使用 secure-storage 加密存储

**解决**: 更新测试逻辑，通过检查 Signal 密钥是否存在来验证注册成功

### 问题 2: 好友添加流程复杂

**发现**: 好友添加涉及多个步骤和 UI 交互，E2E 测试容易超时

**解决**: 将测试拆分为核心功能测试和集成测试，优先验证核心功能
