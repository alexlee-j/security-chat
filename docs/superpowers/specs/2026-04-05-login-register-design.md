# Security Chat - 登录注册页面设计方案

**更新日期**: 2026-04-05
**设计参考**: Telegram Desktop
**主题切换**: Light / Dark 双模式

---

## 1. 概述

本文档定义 Security Chat 桌面端登录和注册页面的视觉设计方案，与聊天页面保持一致的设计语言。

**设计风格**: Telegram Desktop 风格，简洁、专业、安全感

## 4. 组件设计

### 4.1 Logo 区域

```
┌─────────────────────┐
│                     │
│       🔒 SC         │  ← 图标 + 字母缩写
│    Security Chat     │  ← 全称
│                     │
└─────────────────────┘
```

**规格**:
- 图标: 48x48px，圆形背景，主色填充
- 字母: 24px，粗体，白色
- 标题: 18px，常规，文字主色
- 间距: 12px

### 4.2 输入框

```
┌───────────────────────────────────────┐
│  用户名 / 邮箱                        │  ← placeholder
└───────────────────────────────────────┘
```

**规格**:
- 高度: 48px
- 圆角: 12px
- 背景: #f8f9fa (Light) / #1a242d (Dark)
- 边框: 1px solid #dfe1e5 (Light) / #3d4a57 (Dark)
- 内边距: 0 16px
- 字体: 15px

**状态**:
- Default: 上述样式
- Focus: 边框变为主色 (#3390ec / #8777d1)
- Error: 边框变为错误色 (#e53935 / #ef5350)
- Disabled: 50% 透明度

### 4.3 主按钮

```
┌───────────────────────────────────────┐
│              登录 / 注册               │
└───────────────────────────────────────┘
```

**规格**:
- 高度: 48px
- 圆角: 12px
- 背景: 主色 (#3390ec / #8777d1)
- 文字: 白色，15px，粗体
- 居中对齐

**状态**:
- Default: 上述样式
- Hover: 背景变为悬停色 (#2b7fd4 / #9b8cd9)
- Active: 略微缩小 (transform: scale(0.98))
- Loading: 显示加载动画，禁用点击
- Disabled: 50% 透明度

### 4.4 链接文字

```
┌───────────────────────────────────────┐
│        没有账号？注册                  │
└───────────────────────────────────────┘
```

**规格**:
- 字号: 14px
- 颜色: 文字次色 (#707579 / #8b9aa3)
- 悬停: 文字变为主色，可点击

### 4.5 错误提示

```
┌───────────────────────────────────────┐
│  ⚠️  用户名或密码错误                  │
└───────────────────────────────────────┘
```

**规格**:
- 字号: 13px
- 颜色: 错误色
- 背景: 错误色 10% 透明度
- 圆角: 8px
- 内边距: 8px 12px
- 位置: 输入框下方

## 5. 页面流程

### 5.1 登录流程

```
登录页 → 输入用户名/ID + 密码 → 点击登录
                                      ↓
                              验证中 (Loading)
                                      ↓
                         ┌────────────┴────────────┐
                         ↓                         ↓
                    成功 → 跳转聊天页         失败 → 显示错误提示
```

### 5.2 注册流程

```
注册页 → 输入用户名 + 邮箱 + 密码 + 确认密码 → 点击注册
                                                      ↓
                                               验证中 (Loading)
                                                      ↓
                                        ┌─────────────┴─────────────┐
                                        ↓                           ↓
                                   成功 → 跳转聊天页           失败 → 显示错误提示
```

### 5.3 页面跳转

- 登录页底部链接: "没有账号？注册" → 注册页
- 注册页底部链接: "已有账号？登录" → 登录页

## 6. 响应式行为

桌面端窗口最小宽度: 400px

| 窗口宽度 | 卡片行为 |
|----------|----------|
| ≥ 600px | 卡片居中显示 |
| 400-599px | 卡片宽度 90%，居中显示 |
| < 400px | 卡片宽度 100%，贴边显示 |

---

## 7. 字段验证

### 7.1 登录

| 字段 | 验证规则 |
|------|----------|
| 用户名/ID | 必填，最少 3 字符 |
| 密码 | 必填，最少 6 字符 |

### 7.2 注册

| 字段 | 验证规则 |
|------|----------|
| 用户名 | 必填，3-20 字符，字母数字下划线 |
| 邮箱 | 必填，有效邮箱格式 |
| 密码 | 必填，最少 8 字符，包含数字和字母 |
| 确认密码 | 必填，与密码一致 |
